# 我的 OpenWrt 固件方案 (x86_64)

这是一个专为 x86_64 平台（支持 VMware/ESXi 和实体机）优化的 OpenWrt 编译方案。

## 🛠️ 配置亮点

- **架构优化**：针对 x86_64 优化，集成 Intel/Realtek/Broadcom (BCM5719) 等常见网卡驱动。
- **虚拟化支持**：内置 vmxnet3 网卡驱动及 open-vm-tools 组件，完美适配 ESXi 性能和时间同步。
- **主力科学上网**：
  - **OpenClash**：已预置 Meta 内核，支持分流与广告过滤。
  - **SSR Plus+**：经典稳定的代理插件（带 `helloworld` 组件）。
- **常用功能**：原生支持阿里云 DDNS、KMS 服务器激活、UPnP、微信全能推送服务 (PushBot)。
- **网络优化**：集成 Turbo ACC 网络加速。
- **界面定制**：默认采用现代化的 **Argon** 主题。
- **网络网段**：默认主 IP 为 `192.168.3.1`，默认密码为 `password`。

## 📝 编译目的与版本说明

在这个仓库中，每一次启动云编译（GitHub Actions）通常都有明确的目的。您可以通过以下几种方式了解当前版本的变动：

1.  **GitHub Releases 标题**：通常以日期命名，它是您下载固件的地方。
2.  **提交动态 (Commits)**：每一次编译前，我都会通过 Git 提交代码。提交说明里写的内容（如“修复 WireGuard 自动配置”、“修改默认 IP 为 3.1”）就是本次编译的**核心目的**。
3.  **配置文件同步**：通过 `.config` 文件的变化，可以明确本次编译是增加了哪些插件或删减了哪些功能。

## 📂 编译产出文件说明（我该下哪个？）

编译完成后，您会在 Release 页面看到一堆文件，它们的用途如下：

| 文件名 (示例) | 作用 | 推荐使用场景 |
| :--- | :--- | :--- |
| **...combined-efi.img.gz** | **官方首选全量包** | 适用于实体机写盘（用 Etcher/Rufus）、Proxmox、物理机硬盘安装。 |
| **...combined-efi.vmdk** | **虚拟机专用硬盘** | 适用于 **VMware (ESXi / Workstation)**，直接作为现有硬盘挂载。 |
| **config.buildinfo** | 编译配方清单 | 记录了本次编译勾选了哪些插件，方便以后找回配置或对比。 |
| **sha256sums** | 校验完整性 | 用于验证下载的文件是否完整、有没有在传输中损坏。 |
| **...generic.manifest** | 插件列表清单 | 纯文本文件，列出了系统里预装的所有软件包及版本号。 |
| **...rootfs.img.gz** | 根文件系统 | 高级用户使用，用于手动分区或嵌入式系统修复。 |

### 🔍 具体怎么用？

1.  **写盘安装 (U盘/硬盘)**：
    *   下载 `combined-efi.img.gz`。
    *   **必须先解压** 得到 `.img` 文件。
    *   使用 [BalenaEtcher](https://www.balena.io/etcher/) 或 `Rufus` 选择该 `.img` 文件，写入您的 U 盘或硬盘。
2.  **VMware / ESXi 安装**：
    *   下载 `combined-efi.vmdk`。
    *   在虚拟机设置中，添加“现有硬盘”，选择此文件即可（本固件已针对 ESXi 磁盘转换做了优化）。
3.  **验证版本内容**：
    *   如果您不确定固件里有没有某个插件，先下载 `.manifest` 文件搜索一下包名即可，不用急着刷机。

## 🚀 编译图文简单教程

### 第一步：准备编译环境
... (后面保持不变) ...

1. **安装环境**：推荐使用 **Ubuntu 22.04 LTS (或 24.04)** 虚拟机或 WSL2，**请务必使用普通用户**登录，禁止使用 `root`。
2. **迁移文件**：将当前仓库内的所有文件（含 `.config` 等隐藏文件）复制到 Ubuntu 内的某个不存在空格的目录下，如 `~/op/`。
    - **注意！** 在 Linux 终端中，以 `.` 开头的文件是隐藏文件。如果您从 Windows 拷贝文件过去，务必检查是否漏拷了 `.config` 这个关键的配方文件，可以使用 `ls -a` 命令来确认。

### 第二步：首次全量编译

给脚本赋予执行权限，然后一键起飞：
```bash
chmod +x *.sh
./build.sh
```
- 这个脚本会自动帮您拉取最新源码、应用第三方插件（通过 `diy-part1.sh`）、替换主题源及修改 IP（通过 `diy-part2.sh`），然后拉满 CPU 进行全量编译。
- **耗时说明**：第一次编译需要下载很多环境和依赖，通常在 **1 ~ 3 小时** 左右。编译成功后，生成的固件会在 `lede/bin/targets/x86/64/` 目录下。

### 第三步：如何增加新插件并且执行增量更新？

如果您觉得现在的插件不够，想要自己添加或者修改配置，**大可不必重新完整编译**！
1. 在 Ubuntu 内的 `lede` 目录中输入 `make menuconfig` 呼出菜单界面，或者直接修改 Windows 里的 `.config` 文件传进去覆盖。
2. 配置妥当后，回到放着脚本的目录（如 `~/op/`），执行：
```bash
./update.sh
```
- **增量编译极其快速**：它会读取修改后的 `.config`，保留几千个之前编译好的中间件。通常只要 **5 ~ 15 分钟**就能产出新固件！

## 💿 ESXi 部署指南与踩坑说明

由于编译产生的后缀为 `.vmdk` 的固件由于本质上是 Workstation 格式 (monolithicSparse)，在 **ESXi 6.7 等系统**上直接添加硬盘会报错“磁盘大小值无效”。

**如何正确在 ESXi 中安装该固件：**
1. **方案 A（最推荐）**：下载 Windows 免费软件 **StarWind V2V Converter**：
   - 打开软件选择编译出来的 OpenWrt 镜像（选 `.vmdk` 或 `.img` 均可）。
   - 目标格式选择 **"VMware ESX server image"**。
   - 顺带输入您的 ESXi IP 直接传输转换为精简置备的 ESXi 兼容格式。然后在新建虚拟机时“添加现有硬盘”即可。
2. **方案 B（极客选择）**：将刚编译好的 `.vmdk` 上传到 ESXi 硬盘上，通过 SSH 登入 ESXi，利用原生命令转换：
   ```bash
   vmkfstools -i openwrt.vmdk openwrt-esxi.vmdk -d thin
   ```
   然后添加生成出来的 `openwrt-esxi.vmdk`。

## 🔐 WireGuard VPN “极致偷懒”使用指南

本固件已深度定制了 WireGuard 的全开机自动化机制。在您输入密码首次登入后台时，系统其实已经在后台帮您把**所有的服务端私钥、手机端私钥、以至于固定的内网 IP 分配**全部做完了！不需要手动新建节点，更不需要复制这串那串的密钥组合。

**真正的一键式下发连接：**

1. **打开配置表**：刷机并登入后台，进入主菜单中的 **状态 (Status)** -> **WireGuard**（或在 网络 -> 接口 中选择 WG0 查看对端/Peers）。
2. **找到自带节点**：您会惊奇地发现列表中已经为您预置好了两台设备的专属节点：一个是叫 `MyPhone`（为手机备用，IP 定死在 `10.0.0.2`），另一个是叫 `MyPC`（为电脑备用，IP 定死在 `10.0.0.3`）。
3. **填入您的域名**：找到对应的节点，点击最右侧带有二维码形状的 **显示二维码 / 生成配置** 图标。在弹出的配置框中，在端点地址 (Endpoint Address) 里填入您用于外网访问的 DDNS 域名（比如 `myrouter.xyz`）。
4. **一键导入使用**：
   - **📱手机端**：点击生成二维码，拿出手机打开 WireGuard App，选择“从二维码扫描”扫码即得。
   - **💻电脑端**：因为电脑没法扫码，点击生成二维码下方或旁边的 **“保存配置的文本”** 或 **“下载 .conf 文件”**。然后在 PC 版 WireGuard 软件里，点击 **添加隧道 -> 从文件导入**（或新建空隧道复制文本进去）即可。
   
   ✅ **无论是手机还是电脑，私钥、公钥、IP 和服务端域名等所有网络组合全都会瞬间对齐填好！**您唯一要做的就是随手起个名并点击连接！

*注意：确保您的宽带有公网 IP 或者是进行过内网穿透折腾，并且已在光猫和主路由上做好了 udp 协议 `51820` 端口的转发。*
